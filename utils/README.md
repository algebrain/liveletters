# utils

## Назначение

`utils` это корневая папка для общих служебных утилит monorepo LiveLetters.

Ее задача:

- хранить переиспользуемые запускатели;
- хранить общие вспомогательные сценарии;
- не допускать копирования одной и той же служебной логики по подпроектам;
- служить опорным местом для общих команд тестирования, линтинга и форматирования.

## Что здесь должно появляться

- общие сценарии запуска проверок;
- заготовки для локальных запускателей подпроектов;
- вспомогательные функции контроля времени, завершения по ошибке и единообразного вывода;
- служебные оболочки, которые можно вызывать из `test.bb`, `justfile`, `Makefile` или иных локальных команд подпроектов.

## Что здесь не должно появляться

- бизнес-логика приложений и библиотек;
- код, относящийся только к одному подпроекту и не имеющий смысла вне его;
- основная проектная документация.

## Практический принцип

Подпроект может иметь свой локальный запускатель, но этот запускатель по возможности должен быть тонким и опираться на общие утилиты из `utils`.

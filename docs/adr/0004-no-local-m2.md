# ADR 0004: Не использовать локальный `.m2` внутри проекта

## Статус

Принято.

## Дата

2026-04-08

## Контекст

В проекте LiveLetters используется Clojure/ClojureScript tooling, завязанный на Maven-репозиторий зависимостей.

При возникновении локальных проблем с разрешением зависимостей естественным соблазном является создание проектного `.m2` внутри репозитория и перенаправление `deps.edn` на него.

Такой путь в данном проекте нежелателен, потому что:

- он расходится с уже настроенной средой разработки;
- он создает еще одно хранилище зависимостей;
- он делает поведение проекта зависящим от локальной особенности конкретного checkout;
- он размывает единый способ работы с зависимостями.

В текущей среде уже настроено глобальное хранилище зависимостей:

- `/store/m2/repository`


## Решение

В проекте LiveLetters запрещается использовать локальный `.m2` внутри репозитория.

Нужно всегда использовать глобальное хранилище зависимостей, настроенное в среде.

Следствия:

- `deps.edn` не должен задавать проектный `:mvn/local-repo`, указывающий внутрь репозитория;
- внутри репозитория не должна создаваться или поддерживаться папка `.m2` как рабочее хранилище зависимостей;
- все решения проблем с Clojure-зависимостями нужно искать без введения локального `.m2`.


## Почему принято именно это решение

### 1. Единообразие среды

У всех подпроектов должен быть один и тот же источник Maven-зависимостей, а не смесь глобального и проектного.

### 2. Меньше мусора в репозитории

Локальный `.m2` внутри проекта быстро разрастается и делает рабочее дерево тяжелее и менее предсказуемым.

### 3. Меньше скрытой магии

Если проект начинает работать только потому, что внутри него появился особый локальный репозиторий зависимостей, это затрудняет понимание и воспроизводимость среды.


## Последствия решения

### Положительные

- единый способ работы с зависимостями;
- меньше локальных обходных решений;
- чище структура репозитория;
- понятнее настройка среды.

### Отрицательные

- нельзя быстро "починить" проблему зависимостей созданием локального `.m2`;
- иногда потребуется чинить именно окружение или проектную конфигурацию, а не применять локальный обход.


## Правила, следующие из решения

1. Не создавать `.m2` внутри репозитория LiveLetters.
2. Не прописывать в `deps.edn` локальный `:mvn/local-repo`, ведущий внутрь проекта.
3. Использовать глобальное хранилище зависимостей, настроенное в среде.
4. Если Clojure-зависимости не разрешаются, искать причину в конфигурации проекта или окружения, а не вводить локальное хранилище.


## Когда решение нужно пересмотреть

Решение стоит пересмотреть, если:

- базовая среда проекта изменится;
- глобальное хранилище зависимостей будет заменено иной стандартной схемой;
- появятся объективные причины для изолированного локального Maven-хранилища, признанные на уровне отдельного архитектурного решения.

# ADR 0008: В тестах использовать временный `HOME`

## Статус

Принято.

## Дата

2026-04-08

## Суть

- Тесты не должны зависеть от реального домашнего каталога разработчика.
- Если код использует `HOME`, `USERPROFILE` или домашнюю директорию по умолчанию, тест обязан подменять ее временным каталогом.
- Данные тестов не должны попадать в настоящий `~/.liveletters`.

## Контекст

В проекте есть код, который будет хранить данные в домашнем каталоге пользователя, например в `~/.liveletters`.

Если такие тесты опираться на настоящий `HOME`, возникают проблемы:

- тесты перестают быть изолированными;
- тесты могут портить локальное состояние разработчика;
- результаты тестов начинают зависеть от конкретной машины;
- параллельный запуск становится менее предсказуемым.

## Решение

Если тест затрагивает код, зависящий от домашнего каталога, он обязан:

1. создать временный каталог;
2. использовать его как временный `HOME` или эквивалентную домашнюю директорию;
3. работать только внутри этого каталога.

Предпочтительно делать это через явную передачу пути в код теста. Если это невозможно, допустима временная подмена переменных окружения.

Тест не должен читать или писать данные в реальный домашний каталог разработчика.

## Последствия

Плюсы:

- тесты становятся изолированнее;
- тесты безопаснее для локальной машины;
- путь хранения данных проверяется в условиях, близких к реальной работе.

Минусы:

- в тестах появляется дополнительная служебная подготовка;
- для некоторых проверок нужно аккуратно восстанавливать переменные окружения.

## Правила

1. Тест, зависящий от домашнего каталога, обязан создавать временный каталог.
2. Такой тест не должен использовать реальный `HOME` разработчика.
3. Тестовые данные не должны попадать в `~/.liveletters` или аналогичный реальный путь.
4. Предпочтителен явный путь, переданный в тестируемый код; подмена окружения допустима как запасной способ.

## Когда пересматривать

Решение надо пересмотреть, если:

- проект полностью откажется от хранения данных в домашнем каталоге;
- появится более надежная общая абстракция тестового окружения для всех подпроектов.
